# Playing a Wallpaper
The `hypatia play` subcommand takes a pipeline file (a KDL file, typically `pipeline.kdl`) and renders the wallpaper to the desktop. 

For example, you can play the `basic` example in this repository with 

```sh
hypatia play ./examples/basic/pipeline.kdl
```


## Rendering to a specific output
By default, Hypatia renders to the default output that the compositor gives it, which may or may not be the
"primary" display. To change the output the wallpaper gets rendered to, you can a `--output` with an output name, 
for example `--output=eDP-1`.

You can see a list of valid outputs with the `wlr-randr` tool, or by looking
at Hypatia's debug logs by setting `RUST_LOG=debug` in the environment when playing a wallpaper
and see what outputs Hypatia finds. If the specified output by `--output` is not found, Hypatia renders to the default output.
